2/20/2019

Today is February 20, 2019. I have done a lot in the past few days and today's write-up won't be simply about what I've done to this website. I will teach all of you today about the basics on how to use Firebase.

Overview of Firebase:

Firebase is essentially a database that is centered in the cloud. It is a database owned and operated by Google. Firebase is mainly used on applications that on mobile or on a computer for storing global user information. This information could be for authentication while signing in to an application or it could be local app data which is secured in the cloud. There are many features on Firebase for your applications such as holding App data through JSON files, Authentication database for signing on your apps for security, Analytics, and Text Message storing (used by Whatsapp to keep messages sent by users).

What I will teach you today:

Today I will be be going over how you can store TextBoxUI data in Firebase with IOS App Development(Swift).

Step 1:

First Set up Firebase project on their website. Website will be linked in rescources below at bottom of the page. Firebase will give you instructions on installing Firebase to IOS project once you initialize Firebase Project

Step 2:

Once Installed you need to go back to IOS project Folder. You need to open the file that has extension .xcworkspace, because that is where Firebase is installed.

Step 3:

Transfering Data to Firebase

Now is where it gets tricky. For sending only one TextUI to Firebase you need can set the value but for multiple or one data you need to set and ID for each entry. For this I set the Full name of the person in the form as my ID because it is unique so you need to think of one as well. In case you don't understand the code below, it basically is just setting up the JSON tree in Firebase database. So basically In my example I am creating a main branh called Registration where my ID = "FullName" is the child of Registration and then all my TextUI values become childs of "FullName" and so this way with a ID we can keep writing in our TextUI values without overwriting any other data stored in the Registration braanch 

    @IBAction func Sumbit(_ sender: Any) {
        ref.child("Registration").child(FullName.text!).child("Full Name").setValue(FullName.text)
        ref.child("Registration").child(FullName.text!).child("Did Parent Come?").setValue(Did_Parent_Come.text)
        ref.child("Registration").child(FullName.text!).child("Did you wear Uniform?").setValue(Uniform_worn.text)
        ref.child("Registration").child(FullName.text!).child("Phone Number").setValue(Phone_num.text)
        ref.child("Registration").child(FullName.text!).child("Email").setValue(email.text)
        
    }
